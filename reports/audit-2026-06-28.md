## Note méthodologique — headers de sécurité et SEO

L'outil `web_fetch` disponible ne restitue que le contenu rendu et les balises `<meta>`, pas les en-têtes de réponse HTTP bruts, et `robots.txt`/`sitemap.xml` reviennent vides sans code de statut exploitable (limitation outil confirmée depuis trois rapports, pas un signal d'absence). Les vérifications « Headers sécurité » et « SEO » reposent donc sur l'inspection du code source :

- `frontend-web/next.config.js` → `headers()` confirme `Content-Security-Policy`, `Strict-Transport-Security` (avec `preload`), `X-Frame-Options`, `X-Content-Type-Options`, `Referrer-Policy`, `Permissions-Policy`. Inchangé depuis le rapport du 2026-06-22.
- `frontend-web/public/robots.txt` existe, contenu valide (`Allow: /`, exclusions `/admin`, `/account`, `/payment/callback`, `/api/`, référence au sitemap).
- `frontend-web/pages/sitemap.xml.js` existe : route dynamique générant un XML avec dégradation gracieuse si l'API est indisponible.
